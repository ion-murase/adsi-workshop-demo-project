# コードレビュー基準

> テスト要件は [../testing.md](../testing.md)、セキュリティは [../security.md](../security.md) を参照。

## レビューが必須なタイミング

- コードを書いた・変更した直後
- 共有ブランチへのコミット前
- 認証・認可・ユーザー入力処理を変更したとき
- アーキテクチャを変更したとき
- PR マージ前

## レビュー前の確認

- CI / ローカルチェックが通っている（`npm run check:backend`, `npm run lint:frontend` 等）
- マージコンフリクトが解消されている
- ベースブランチと同期されている

## チェックリスト

- [ ] 読みやすく、命名が適切
- [ ] 関数が焦点を絞っている（<50 行）
- [ ] ファイルが凝集している（<800 行）
- [ ] ネストが浅い（>4 レベルになっていない）
- [ ] エラーが明示的に処理されている
- [ ] シークレットのハードコードがない
- [ ] デバッグ用の `console.log` 等が残っていない
- [ ] 新機能にテストがある
- [ ] カバレッジ 80% 以上（目標）

## 重大度

| レベル | 意味 | 対応 |
|--------|------|------|
| CRITICAL | セキュリティ脆弱性・データ損失リスク | マージ前に必ず修正 |
| HIGH | バグ・重大な品質問題 | マージ前に修正推奨 |
| MEDIUM | 保守性の問題 | 修正を検討 |
| LOW | スタイル・軽微な提案 | 任意 |

## セキュリティレビューが必要な変更

以下を触ったときは [../security.md](../security.md) を重点確認する:

- 認証・認可
- ユーザー入力処理
- DB クエリ
- ファイル操作
- 外部 API 呼び出し

## よくある問題

### セキュリティ

- ハードコードされた認証情報
- SQL インジェクション（文字列結合クエリ）
- XSS（エスケープされていないユーザー入力）

### 品質

- 大きすぎる関数・ファイル
- 深いネスト
- エラーハンドリング不足
- テスト不足

### パフォーマンス

- N+1 クエリ
- ページネーション不足
- 無制限クエリ

## 承認基準

- **Approve**: CRITICAL / HIGH なし
- **Warning**: HIGH のみ（慎重にマージ）
- **Block**: CRITICAL あり

## Multi-Agent Review（推奨）

並列 subagent レビューは **`multi-agent-review` スキル** を使う:

- `java-reviewer`, `typescript-reviewer`, `security-reviewer`, `test-reviewer` を並列起動
- subagent は **`model: sonnet`**（`.claude/agents/*.md` で設定）
- 詳細: `.claude/skills/multi-agent-review/SKILL.md`
